# 个人版架构说明

## 已采用的方案

个人版删除登录等待、多人在线、权限角色和公司 Appwrite 运行依赖。网页是静态 React 应用，素材数据库是浏览器 IndexedDB。

```text
小红书 / 花瓣 / Pinterest
          ↓ Chrome 插件读取媒体字节
个人灵感库网页
          ↓
IndexedDB：原图 Blob + 720px WebP 预览图 + 标题/标签/来源
          ↓
完整备份：manifest.json + media/原始文件
```

## 缩略图是否会拖慢或占很多内存

上传时确实需要短暂解码图片，但实现为逐张串行处理，不会让多张大图同时占用解码内存。每张完成后立即释放 `ImageBitmap`。长期额外占用的是一张最长边 720px、质量 0.78 的 WebP 预览图，通常远小于原图；换来的收益是图库滚动不再反复解码原始大图。

## 原图在哪里

原图和预览图分开保存。卡片用预览图，点开详情使用原始 Blob，“下载原图”和导出备份也都读取原始 Blob。不会用 720px 预览图覆盖原文件。

## 换设备怎么解决

第一阶段用“导出/导入”：导出目录中 `media/` 是全部原文件，`manifest.json` 是标题、标签、文件夹、来源等元数据。复制这个目录到新设备后导入即可完整恢复。

若需要真正自动同步，必须增加一个设备之间共同访问的存储端。可以是 NAS，也可以是阿里云 OSS、腾讯云 COS、S3/R2 等对象存储；同时还需要一个很小的同步服务保存元数据并签发短期上传地址。不能把云存储永久密钥直接写进网页或插件。

## 登录是否保留

当前单人本地版没有必要保留登录。等启用云同步时，再增加“设备令牌/通行密钥”即可；只有要分享给别人或多人协作时，才值得恢复完整账号体系。

## 本地视觉功能与图片反推

色板和色相使用浏览器 Canvas 在本机计算，新图片上传时立即生成；旧数据在图库打开后后台串行补齐，不阻塞首屏。一键图片反推保留中英文提示词、设计术语、氛围词和搜图词，但改为用户自备公网视觉模型 API，不再依赖快手万擎或公司 Appwrite Function。分析只发送 720px 预览图，结果写回 IndexedDB 并随完整备份导出。
